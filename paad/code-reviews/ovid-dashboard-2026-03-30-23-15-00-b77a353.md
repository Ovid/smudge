## Important Issues

### [I1] Trash view persists when switching to Preview or Dashboard tabs

- **File:** `packages/client/src/pages/EditorPage.tsx:473` (render), `:427-453` (buttons), `:179-185` (keyboard)
- **Bug:** The render logic `trashOpen ? <TrashView> : viewMode === "preview" ? ...` gives `trashOpen` unconditional priority over `viewMode`. The Preview and Dashboard button onClick handlers change `viewMode` but never clear `trashOpen`. The Ctrl+Shift+P keyboard shortcut also fails to clear `trashOpen`. When trash is open and the user clicks Preview/Dashboard or presses Ctrl+Shift+P, the tab indicator updates (via `aria-current`) but the content area continues showing trash.
- **Impact:** The UI enters an inconsistent state -- the active tab says "Dashboard" or "Preview" but the content shows "Trash". The `aria-current` attribute lies to assistive technology. The only escape is clicking "Back to editor" inside the trash view.
- **Suggested fix:** Add `setTrashOpen(false)` to the Preview button onClick (line 427), Dashboard button onClick (line 441), and the Ctrl+Shift+P handler (line 181). Alternatively, guard shortcuts with `if (trashOpen) return;`.
- **Confidence:** High
- **Found by:** Logic & Correctness, Concurrency & State

### [I2] Stored XSS vector via `dangerouslySetInnerHTML` in PreviewMode

- **File:** `packages/client/src/components/PreviewMode.tsx:69`
- **Bug:** `dangerouslySetInnerHTML={{ __html: html }}` renders HTML generated by TipTap's `generateHTML()` from stored JSON content without sanitization. While TipTap's standard nodes escape text, the `TipTapDocSchema` in `shared/schemas.ts` uses `.passthrough()` and accepts `z.record(z.unknown())` for content nodes, meaning arbitrary JSON structures are accepted by the API. If malicious TipTap JSON is written to the database (via the unauthenticated PATCH endpoint), it could produce executable HTML.
- **Impact:** In the current single-user, no-auth context, the attack surface is limited but real -- any script on the same origin, browser extension, or local network attacker can write to the API. Defense-in-depth principle applies.
- **Suggested fix:** Sanitize with DOMPurify: `dangerouslySetInnerHTML={{ __html: DOMPurify.sanitize(html) }}`. Alternatively, tighten `TipTapDocSchema` to allowlist known-safe node types.
- **Confidence:** Medium
- **Found by:** Security
