## Important Issues

### [I1] `URL.revokeObjectURL` called before browser processes download
- **File:** `packages/client/src/components/ExportDialog.tsx:91-98`
- **Bug:** The download trigger sequence calls `a.click()` then immediately `URL.revokeObjectURL(url)` in the same synchronous block. Firefox and Safari initiate downloads asynchronously after `click()` returns — revoking the object URL before the download begins can cause the download to silently fail (empty file or no download at all).
- **Impact:** Export downloads may silently fail on non-Chromium browsers. The dialog closes (line 100: `onClose()`) but no file appears. No error is shown to the user.
- **Suggested fix:** Defer revocation with `setTimeout(() => URL.revokeObjectURL(url), 0)` or a small delay (100ms).
- **Confidence:** High
- **Found by:** Error Handling & Edge Cases, Concurrency & State

## Suggestions

- `packages/server/src/export/export.renderers.ts:165,169,193` — Markdown title, author, and chapter titles are not escaped for Markdown metacharacters (`*`, `_`, `[`, `` ` ``). A title like `My *Book*` produces unintended italic formatting; a title with a newline breaks the heading structure entirely. Consider adding a `escapeMd()` helper for plain-text strings used in Markdown headings/emphasis. (Logic & Correctness, Security)

- `packages/server/src/export/export.renderers.ts:57-63` — `stripHtmlTags()` only decodes 6 HTML entities (`&amp;`, `&lt;`, `&gt;`, `&quot;`, `&#39;`, `&nbsp;`). TipTap's `generateHTML()` may produce named entities like `&mdash;`, `&ndash;`, `&hellip;` and numeric entities like `&#8212;` — these appear as literal entity strings in plaintext export. Consider adding a general numeric entity decode pass. (Error Handling & Edge Cases)

- `packages/server/src/export/export.renderers.ts:175-183` — Markdown TOC anchors generated by `slugifyAnchor()` assume GFM-compatible heading-ID generation. Different Markdown renderers (Pandoc, CommonMark, VS Code) use different algorithms, so TOC links may be non-functional in some viewers. Consider adding a comment noting GFM-tuning, or using index-based anchors like the HTML renderer. (Logic & Correctness, Contract & Integration)

- `packages/client/src/api/client.ts:114-131` — The export API method hand-rolls the same error-extraction pattern as `apiFetch` (check `!res.ok`, try parse JSON, extract `error.message`). If error format or retry logic changes in `apiFetch`, the export path won't benefit. Consider extracting a shared `handleErrorResponse` helper or `apiFetchBlob` variant. (Contract & Integration)

- `packages/client/src/components/ProjectSettingsDialog.tsx:177-180` — `handleDeadlineChange` calls `saveField` directly on every `onChange` without an AbortController, unlike `handleTimezoneChange` which cancels previous in-flight requests. Rapid date changes via keyboard arrows could cause out-of-order PATCHes. (Concurrency & State)

- `packages/server/src/export/export.renderers.ts:193` + `packages/shared/src/schemas.ts:55` — `UpdateChapterSchema` allows internal newlines in titles (`z.string().trim().min(1).max(500)` — `trim()` only strips leading/trailing whitespace). A title with `\n` would break Markdown headings (`## Title\nrest`) and corrupt plain text TOC structure. (Logic & Correctness)

- `packages/server/src/export/export.renderers.ts:128` — `chapterContentToHtml()` output is injected verbatim into the HTML export template. In the single-user, TipTap-authored context there is no realistic attacker, but as defense-in-depth, consider sanitizing generated HTML before template injection (e.g., via DOMPurify) for forward safety. (Contract & Integration)
